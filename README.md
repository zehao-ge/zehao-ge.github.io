# Ge Zehao — academic personal website

A static academic personal website for embodied-AI, robot-learning, and human–robot interaction research.

## Update content

All visible copy, metadata values, accessibility labels, structured-data content, links, and TODOs live in:

```text
src/content/site.ts
```

Update that file rather than editing strings inside components.

## Local development

Requirements: Node.js 22.13 or newer.

```bash
npm install
npm run dev
```

Open `http://localhost:3000`.

## Production build

```bash
npm run build
```

The fully static export is written to `out/`. It can be served from any static host.

## Deploy to Vercel

1. Import this repository into Vercel.
2. Use the Next.js framework preset.
3. Keep the build command as `npm run build`.
4. Deploy. Vercel will serve the static export generated by `output: "export"`.
5. Replace the canonical URL TODO in `src/content/site.ts` with the final domain.

## Deploy to GitHub Pages

1. Push the repository to GitHub.
2. Configure a GitHub Actions workflow to run `npm ci` and `npm run build`.
3. Upload the `out/` directory with `actions/upload-pages-artifact`.
4. Deploy it with `actions/deploy-pages`.
5. If deploying under a repository subpath, add matching `basePath` and `assetPrefix` values to `next.config.ts` before building.
6. Replace the canonical URL TODO in `src/content/site.ts`.

## Deploy to Cloudflare Pages

Use `npm run build` as the build command and `out` as the output directory.
